# Zettlab — Design Context

> **Last updated:** 2026-04-22
>
> 给所有未来的设计/开发决策提供"为什么"的锚点。

## Users

**主画像：专业创作者 / 开发者**
- 设计师、独立开发者、内容创作者
- 对 AI 工具有实际工作需求，追求效率和产出质量
- 审美参照：高品质工具产品的活跃用户

**辅画像：数字生活追求者**
- 热爱新产品、愿意为设计买单、会在社交媒体分享产品截图
- 不一定深度技术，但有审美洞察力

**使用场景：**
- 移动端为主（mobile-first），home cloud AI Agent 平台
- 从 onboarding 到 workspace 的完整用户旅程
- 当前为高保真 demo，后续迭代为可对外展示的设计稿规范

## Brand Personality

**三个关键词：清晰 · 克制 · 温度**

### 三词展开

- **清晰（Clarity）**：界面为内容让路，层级通过光影微妙变化自然浮现，不依赖装饰性元素
- **克制（Restraint）**：信息密度高但视觉噪音低，通过减少可见的设计决策来降低认知负担
- **温度（Warmth）**：精密不等于冷峻，字重略厚、色温略暖、圆角略柔，让工具有人情味

### 声音 / 语气

- 专业但不冷峻
- 克制但不寡淡
- 精确但有温度
- 像一个"有品位的资深朋友"，不是教科书也不是广告

### 情感目标

用户应该感受到：**信赖、审慎、被尊重、偶尔的小惊喜**。
用户不应该感受到：浮夸、廉价、拥挤、粗糙、无个性。

## Aesthetic Direction

### 视觉基调

- **Dark-first，双模式完整定义**：深色是主场，浅色是完整对应（非阉割版）
- **单一有彩色**：Warm Indigo `#6366DE`（深色）/ `#5350D2`（浅色），其余全灰阶
- **色温分区**：深色的灰阶偏冷骨架 + 文字微暖点缀；浅色的灰阶全链路偏暖

### 设计取向（正向）

| 方向 | 取什么 |
|---|---|
| **冷调精准骨架** | 深色骨架、surface 亮度阶梯、Inter Variable + cv01/ss03、三档字重制、半透明白边框 |
| **暖调温润触点** | 浅色暖调、Warm Charcoal 文字、Parchment 暖灰边框、字重 450 的温润、极简圆角（8/16） |
| **紧凑信息密度** | 紧凑布局的信息密度参照（不照搬暖色点缀） |
| **流畅动效节奏** | 动效节奏、呼吸感 panel 切换（将来 Step 4 动效参考） |

### 反参考（明确排除）

- ❌ IoT 平台风：多彩圆角卡、拟物图标、"智能家居"廉价感
- ❌ SaaS 仪表盘风：数据密集、多彩 chart、信息量爆棚
- ❌ 游戏 / 赛博朋克风：霓虹、发光线条、激进动效、RGB 配色
- ❌ 极简白 / 无个性风：过度克制到没有个性，纯白 + 纯灰，毫无品牌轨迹
- ❌ 当前 demo 的状态：粗糙、缺细节、不精致、不统一、没规范

### 无障碍

- WCAG AA 为基线（文字对比度 ≥ 4.5:1）
- 尊重 `prefers-reduced-motion`（已在 globals.css 实现）
- 关键交互有非色彩区分（不能只靠颜色传达状态）

## Design Principles

### 01 Clarity（清晰）

界面为内容让路。层级通过光影的微妙变化自然浮现，而非依赖装饰性元素。

- 深色模式靠 background 亮度阶梯（0.02 → 0.04 → 0.05 → 0.07 四级透明度）表达深度，不靠阴影
- 浅色模式靠暖灰边框 + 极轻阴影表达层级
- 只有 `brand`（Warm Indigo）是有彩色，其余全灰阶；status 色只用于语义传达，不用于装饰

> *成功标准：用户不会有意识地注意到设计本身。*

### 02 Restraint（克制）

信息密度高，但视觉噪音低。通过减少可见的设计决策来降低认知负担。

- 禁用 weight 700+、禁用饱和装饰色、禁用重阴影、禁用 2/6/22px 等非体系圆角
- 所有"加一点"的冲动，先问"能不能减一点"
- 用户感知到秩序，但感知不到规则本身

> *判断标准：加元素前问"减掉这个会让功能失效吗？"。如果不会，就减掉。*

### 03 Warmth（温度）

精密不等于冷峻。字重略厚一点，色温略暖一点，圆角略柔一点。

- 三档字重（450 / 510 / 590）承载 90% 场景，body 默认 450 而非 400
- 色温分区：深色灰阶偏冷骨架 + 文字微暖；浅色灰阶全链路偏暖
- 圆角下限 8px，工具应该像一个可靠的伙伴，而非冰冷的机器

> *判断标准：当你质疑"这里是否太冷"时，去加一个温润触点（色温/字重/圆角），而不是改骨架结构。*

## Do's and Don'ts

### Do

**字体 & 排版**
- 全局开启 Inter Variable 的 `cv01` + `ss03`
- body 用 `font.weight.body` (450)，不用 400
- 标题按 size 用递增负字距（display -0.024em → headline -0.008em）
- 压缩标题行高（1.05-1.25）+ 透气 body 行高（1.50）

**颜色**
- 主文字用 `text.primary`（深色 `#F7F6F3` / 浅色 `#2A2825`），永远不用纯白/纯黑
- Brand 色只用在交互/CTA/激活态/链接
- 语义色只用于语义传达，不做装饰
- 深色边框用 `border.*` 半透明白，浅色用暖灰

**深度 & 布局**
- 深色模式用 surface 亮度阶梯表达层级
- 浅色模式用 border + 轻阴影表达层级
- 圆角只用 `radius.*` 5 档，不用非体系值
- 间距只用 `space.*` 13 档，不用 5/7/11 等奇数值

**组件**
- Agent avatar 用方形圆角（`radius.md` 8px），区分人的圆形
- Tab bar 激活态用 `text.brand`，未激活用 `text.tertiary`
- 消息气泡用 radius 18px + 对侧角 4px（显示归属感）

### Don't

**字体 & 排版**
- 不用 `font-weight: 700+`（超出克制气质上限）
- 不跳过 `cv01` + `ss03` OpenType 特性
- body 字重不用 400 默认值（必须 450）
- 不在展示字号用正字距

**颜色**
- 不用 `#FFFFFF` 作主文字（刺眼）
- 不用 `#000000` 作浅色模式文字（在暖底上冷得突兀）
- 不在按钮用饱和色（除 primary 的 brand）
- 不把 brand 紫拿来装饰分割线/图标底/标签背景
- 不引入橙/粉/黄做品牌色装饰（semantic 除外）

**深度 & 布局**
- 不在深色背景上用 `filter: drop-shadow()` 或重阴影
- 不用 `radius: 2/6/22/24px` 等非体系值
- 不用 5/7/11/13px 等非网格间距
- 不用 `border: 1px solid #333` 等实线暗灰（深色下会显脏）

**组件**
- 不给人物头像用方角
- 不给 agent 头像用圆形
- 不在 chip/badge 使用 `radius.sm` 以下圆角（最小 `radius.full`）
